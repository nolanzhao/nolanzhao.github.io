2026 年 5 月 13 日，特朗普踏上中国。一个看起来很无聊的问题被一个 YouTuber 玩成了节目：他在中国用什么 VPN？

这个问题表面是猎奇，本质是个非常好的引子，能把以下几个完全不同的领域串起来：

  • 美国总统的国家级通讯体系（WHCA / NSA / 核足球）
  • 端点安全、链路安全、端到端加密这三个最容易被混淆的术语
  • GFW 的设计假设和它对单点用户的"无效"
  • 一段 1945 年的间谍故事，是现代信息安全学科的开山案例
  • 维也纳公约里关于加密通讯权的条款
  • 苹果 FaceTime 在协议层到底做了什么
  • 为什么"和总统一样的安全级别"这种营销词永远是错的

一、特朗普访华的"VPN"是什么

先把 VPN 这个词拆清楚。

  • 对中国普通用户：VPN ≈ 翻墙工具，目的是把流量从中国运营商网络偷偷送出去
  • 对美国总统：VPN = 从所在国家任何不可信网络，安全地接回美国本土

后者的解决方案叫 WHCA（White House Communications Agency，白宫通讯局）。

  • 1942 年成立，前身是 White House Signal Detachment
  • 编制约 1000 人，跨海陆空三军 + 海岸警卫队 + 海军陆战队的联合机构
  • 现归国防部下属的 DISA 管理
  • 使命只有一句话：确保美国总统在世界上任何一个角落都能跟华盛顿通话，不被任何人监听

1.1 访华的工作流程

特朗普到北京之前一周，先遣队就会飞到目的地（这次住四季酒店，后面会讲为什么）。先遣队做的事：

  1. 用专业设备扫描整栋楼，找窃听器
  2. 从屋顶拉一根光纤下来，接到临时部署的军用卫星地面站
  3. 所有窗户加装振动干扰器，或换成防激光窃听玻璃
  4. 在总统卧室和会议区搭一顶 SCIF（Sensitive Compartmented Information Facility）—— 物理上就是"帐篷里的帐篷"，金属编织墙 + 缝隙做电磁屏蔽。手机进去没信号，外面听不到里面的对话

媒体估算这次访华光后勤就 2100 万美元，包括防弹车、通讯车、医疗救护车、移动通信基站、卫星终端、集装箱式临时指挥中心。

1.2 数据链路

特朗普的设备发出的每一个数据包，走的链路是这样的：

[设备]
   ↓
[Taclane 加密器]   ← NSA Type 1 认证密码机（通用动力出品）
   ↓
[屋顶光纤]
   ↓
[军用同步轨道卫星 X-Band UHF]   ← 不走 Starlink，不走任何民用通讯
   ↓
[Camp David WHCA 交换机]
   ↓
[美国境内公网]

关键观察：数据包从未进入中国的电信运营商网络。

GFW 的所有手段——DNS 污染、IP 黑名单、SNI 嗅探、TLS 指纹识别、主动探测——全部建立在一个前提上：流量得经过中国境内的路由器。特朗普直接走天上，绕过去了。

更准确地说，GFW 对他来说不是"被绕开"，而是完全不存在。

1.3 GFW 的不对称性

这条思路引出一个有意思的推论。上一期那个 YouTuber 讲过快连为什么被封：因为它有几十万付费用户的流量池，全部强制经过中国 ISP 出境。这是 GFW 最强的场景。

套用同样的逻辑，为什么特朗普封不动：

  • 单人单流量
  • 不经过中国网络
  • 国家级加密
  • 还有外交豁免权（后面讲）

GFW 的每一个设计前提，在他身上都不成立。这就是 GFW 的不对称性：

维度 GFW 的表现
对穷人 非常不友好
对富人/国家级用户 无效
对规模化用户 极其精准
对单点用户 无力
对境内流量 极度有效
对绕开境内的流量 完全失效

二、总统身上的四类设备

2.1 未改装的私人 iPhone

完完全全跟你我手上一样的 iPhone，连麦克风都没拆。这是特朗普的任性，也是 NSA 最头疼的地方。

多位现任和前任美国官员证实：俄罗斯和各国情报机构都在常规监听特朗普的私人 iPhone。这不是猜测，是美国情报系统的内部评估。

2.2 NSA 改装的 iPhone（两台冗余）

外壳是 iPhone，但内部被砍掉一大批功能：

  • 第三方 App Store
  • iCloud 同步
  • iMessage
  • Find My iPhone
  • AirDrop
  • 个人热点
  • 摄像头/麦克风常驻权限
  • GPS 精准定位
  • 蓝牙发现模式

留下来的几乎全是网关：

  • 电话 → 走 D2D 加密语音网关
  • 短信 → 走专用安全通道
  • Truth Social / X → NSA 专门立项批准（后面讲那个搞笑故事）
  • 邮件 → 阉割版客户端只能连 Outlook
  • 日历

两台冗余，号码定期轮换，当前号码的知情人不超过 20 个。

2.3 Cisco 8841 改装版 IP 电话（黄色边框）

这台机器有几个非常硬核的改造：

① 黄色边框 = 视觉防呆

WHCA 的颜色编码：

  • 绿色 = 非密级
  • 黄色 = 机密级
  • 红色 = 绝密级

在错误的网络上讨论错误内容，是最高级别的安全事故。颜色编码是物理性的防呆，和数据中心用不同颜色网线区分 VLAN 是一个思路。

② 光纤上联，取代 RJ14 铜缆

为什么？铜缆传输的电信号会通过电磁辐射泄露到周围空间。1～3 米内用普通 SDR 接收机就能还原原始音频。光纤是光信号，物理上没这个旁路。

③ TEMPEST 屏蔽外壳

所有缝隙做电磁屏蔽，麦克风听筒部分做法拉第笼处理。

④ On-Hook Disconnect（挂机即断开）

这是最关键的安全特性。普通办公电话挂机后麦克风仍然带电，理论上可以被远程激活当窃听器，叫 Hotmic 攻击。

这台改装版在挂机时物理切断麦克风电源——不是软件控制，是机械继电器。

⑤ 终端傻、网络聪明的设计哲学

电话本身不加密。加密发生在中间的 Taclane（KG-175D/G）密码机里，那才是真正用 Type 1 密码加密的地方。Cisco 8841 只是把模拟语音转成 IP 包丢到光纤上。

这种思路是冷战时期确立的：把保密的复杂度集中在少数高度受控的硬件里，终端可以用任何东西，方便量产和换代。

2.4 核足球（Nuclear Football）

7×24 不离身。洗澡时核足球军官在浴室门口待命，做肠镜麻醉时副总统代理职权但军官在隔壁。

总统年薪 40 万美元固定（宪法第二条第一款规定，任期内不得加薪也不得减薪），没有 KPI 考核。核足球岗位是地球上极少数没有"下班"概念的工作之一。

三、端点安全 / 链路安全 / 端到端加密

这是整个话题里最容易被混淆的三个概念，也是理解后面所有内容的基础。

术语 保护谁 怎么破
端点安全 设备本身（系统、芯片、App） 攻破你的手机/电脑
链路安全 中间传输路径（基站、骨干网、海底光缆） 控制中间任何一段
端到端加密（E2EE） 内容本身（在你设备上就已加密） 必须攻破两端设备之一

视频里那句"苹果端点做得很好，但链路苹果管不着"——意思是：

  • 苹果能保证 iPhone 本身没漏洞（端点）
  • 但你的电话信号一旦出了 iPhone，走的是运营商网络、走的是 1970 年代设计的 SS7 协议
  • 这条链路苹果设计不了、也控制不了

只有 iMessage / FaceTime 这种 E2EE，内容在你 iPhone 里就被加密了，中间链路即使被全程监听，看到的也是乱码。

简单记：

  • 端点 = 起点和终点的设备
  • 链路 = 起点到终点之间的整条路
  • E2EE = 让链路再脏也无所谓

四、SS7 漏洞：为什么打普通电话就是裸的

SS7 = Signaling System 7，1970 年代设计的电信信令协议。全球电信网络至今依赖它来做漫游、来电显示、短信路由。

著名漏洞：知道一个手机号，就能定位、转接、监听这个号码的所有通话和短信。

跟你用什么手机无关——iPhone、华为、诺基亚老人机都一样。漏洞不在手机里，在电信骨干网里。

各国情报机构怎么监听特朗普？大概率根本不需要破解他的 iPhone，只需要：

  1. 买通某个小国的电信运营商（中东 / 非洲 / 东南亚都行）
  2. 获得 SS7 网络接入权
  3. 发送伪造信令请求，假装特朗普的手机正在该国漫游
  4. 所有去往特朗普号码的通话先转接到中间节点，再转回原目的地

经典的中间人攻击。特朗普和对面都察觉不到。

反直觉的结论：苹果做得很好，但只是端点安全。链路安全苹果管不着，因为电信骨干网不是苹果建的。

特朗普如果跟你我一样喜欢打普通电话而不用 iMessage/FaceTime，那么特朗普的通讯，至少在国家机器面前，是裸的。这不是苹果的错，也不是 NSA 没尽力，是每个人个人的选择——把自己暴露在一个 1970 年代设计的协议漏洞之下。

五、FaceTime 的工作原理

既然 SS7 这么脏，那 FaceTime 凭什么"安全"？拆开看：

你按下拨号
  ↓
1. 信令：通过 APNs + IDS 把"我要呼叫你"推给对方
  ↓
2. 密钥协商：双方通过 IDS 拿对方公钥，ECDH 算出共享密钥
  ↓
3. NAT 穿透：ICE/STUN 探路，理想情况两台设备直连
  ↓
4. 媒体传输：音视频走 SRTP，用上面那把共享密钥加密

5.1 IDS（Identity Service）= 苹果的"公钥目录"

每台设备首次激活时，本地生成一对 ECDH（P-256）非对称密钥，私钥永远留在设备 Secure Enclave 里，公钥上传到苹果 IDS 服务器。

IDS 把"手机号/邮箱 → 一组设备公钥"建成索引。你的 Apple ID 同时登录 iPhone + iPad + Mac，IDS 里就有 3 个公钥条目。

5.2 APNs + 信令

呼叫发起：

  1. 你的设备问 IDS：“对方所有设备的公钥给我”
  2. 通过 APNs（推送服务）推送一条加密信令到对方所有设备（响铃就是这个）
  3. 信令里包含 SDP（编解码器、ICE 候选地址）和你的临时会话公钥
  4. 对方接听 → 反向推回它的会话公钥
  5. 双方各自做 ECDH → 算出一把只有这次通话有效的共享密钥

苹果服务器从头到尾只是个信箱，没参与算密钥。

5.3 ICE / STUN / TURN —— 穿透 NAT

家庭路由器后面的设备没有公网 IP，直接发包对方收不到。FaceTime 用 IETF 标准的 ICE 解决：

  • STUN：问苹果的 STUN 服务器"我在公网上看起来是什么 IP:端口?"
  • 双方交换所有可能的地址候选（局域网 IP / 公网映射 IP / 中继 IP）
  • ICE 算法挨个尝试连通性，选最优一条

大约 80% 的 FaceTime 通话能做到 P2P 直连（媒体流根本不经过苹果服务器）。

剩下穿透不了的（对称 NAT、企业防火墙）走 TURN 中继：苹果服务器做"傻包转发"，仍然看不到内容——因为内容已经被 SRTP 加密。

5.4 媒体传输：SRTP

  • 加密：AES-CTR / AES-GCM
  • 用 5.2 算出的会话密钥（每次通话不同，具备 Perfect Forward Secrecy）
  • 编解码器：音频 AAC-LD，视频 H.264 / H.265 / 苹果自家 LCEVC
  • 每个 RTP 包都加密 + MAC 防篡改

5.5 为什么说苹果"看不到内容"

  • 私钥从未离开设备（Secure Enclave 硬件保护）
  • 共享密钥是两台设备 ECDH 算出来的，苹果服务器没参与运算
  • 媒体流在 P2P 模式下根本不过苹果服务器；过的话也是加密包
  • 即使法庭传票来，苹果能交出的只有"几月几号 A 呼叫了 B"这种元数据，没有通话内容

5.6 已知的"理论后门"

这是 E2EE 最常被讨论的弱点，不只 FaceTime，所有 E2EE 系统都有。

苹果控制 IDS 公钥目录。 理论上，苹果可以在你呼叫某人时，偷偷往返回的公钥列表里插入一台幽灵设备的公钥（政府/苹果控制），你会无察觉地把会话密钥也加密给它。

这叫 Ghost User 攻击。FBI 在 2018–2020 年间公开提过对苹果的这种要求，苹果对外否认配合，但密码学上只能信"苹果说不"——没法从协议本身证伪。除非将来苹果引入 Signal 那种"公钥安全编号"（让双方手动核对指纹），但那不太可能，因为太破坏用户体验。

5.7 跟普通电话的对比

普通电话 FaceTime
信令网络 SS7（1970s，有漏洞） APNs + IDS（苹果自家）
寻址 手机号→运营商 HLR Apple ID→IDS
加密 无（明文走骨干） E2EE（SRTP + ECDH）
中间人 任何运营商 / SS7 攻击者都行 只有苹果（理论上的 ghost key）
元数据 运营商全留 苹果保留呼叫双方+时长

六、1945 年的木雕国徽：现代信息安全的开山故事

这次访华，特朗普为什么住北京四季酒店而不是钓鱼台国宾馆？

不是因为携程房间订满了。钓鱼台是主权空间，管理权和所有权全在中方。四季酒店是商业空间，美方可以包楼接管，把它临时转化成"美方控制的飞地"。加上离美国大使馆只有 700 米，万一有事可以快速进入享有治外法权的主权领土。

这是"最小可信空间“原则的应用，和总统卧室里那顶 SCIF 帐篷是同一个思路。

那为什么 WHCA 要把酒店里所有的家具都扫一遍？因为有 1945 年那个故事。

6.1 故事

1945 年二战刚结束，美苏蜜月期。苏联少先队的孩子们到莫斯科的美国大使官邸，送给当时的大使 Averell Harriman 一个手工雕刻的美国国徽作为友谊礼物。

Harriman 很高兴，挂在书房墙上，正对办公桌。

接下来 7 年间，这栋楼住过 4 任美国大使。所有人都在书房里讨论过机密：电报草稿、跟华盛顿的电话、跟其他外交官的密谈。全部被苏联监听了 7 年。

1952 年才意外被发现：英国一个驻莫斯科电台的无线电操作员，碰巧在某个频段听到了美国大使的声音，他认得这个声音，但完全不知道为什么会出现在那个频段上。报告给英方，英方又告诉美方。

美方派人去这个房间做了三次完整的电子扫描，都没找到——因为这个设备绝大部分时候不发任何信号。

后来 NSA 想了一个新办法：在房间里故意制造各种音频，同时用宽频接收机扫描。发现当且仅当外部某个特定频率（约 330 MHz）的微波从外面射进来的时候，房间里的语音会被调制到反射回去的信号上。顺着这个原理，才找到了木雕国徽里的装置。

6.2 设计

设计者叫 Leon Theremin——就是发明特雷门琴的那个 Theremin。俄裔美籍发明家，二战期间被苏联从美国召回为克格勃服务。

装置的原理：

  • 一个谐振腔 + 短天线 + 金属薄片（作为麦克风的共振膜）
  • 完全无源：没有电池、没有电源、没有任何主动电子元件
  • 不被照射的时候就是一坨金属，X 光都很难看出问题
  • 工作时苏联在街对面停一辆面包车，发射 330 MHz 微波束照射大使馆
  • 微波激发谐振腔 → 房间里的声波震动金属薄片 → 薄片改变谐振腔的电气特性 → 反射回去的微波被声音调制

这一类窃听器，是现代 RFID 技术的直接祖先。

6.3 美国的反应

发现之后美国选择忍气吞声，没声张。

直到 1960 年 U-2 间谍机被苏联击落，美方在联合国安理会想谴责苏联间谍行为，苏方反击说"那不是美国也搞间谍吗”——美国大使当时就掏出了这个木雕国徽的复制品，抛给全世界看。

等于公开宣告：“看看你们干的好事。”

这是冷战间谍战公开亮相的标志性时刻。

6.4 它确立的现代信息安全原则

为什么这个故事到今天还要讲？因为它确立了几个核心原则：

① 物理空间也是攻击面

网络可以是零信任的，但房间有窗户、有墙、有装饰品、有暖气片——每一个都是潜在的旁路信道。

② 被动设备无法用常规电子扫频检测

直接导致 TSCM（Technical Surveillance Counter-Measures，技术监听对抗）这门专业的诞生。

③ 任何礼物都可能是攻击向量

美国国务院从此规定：驻外使馆收到的所有礼物必须经过专业扫描，才能带入工作区。这条规定至今有效。

所以特朗普这次先遣队在四季酒店做的那一周准备里，大概率包括现代版的 NLJD（非线性结点探测器），扫描所有家具、装饰品、墙体，寻找现代版的 The Thing。

七、维也纳公约第 27 条

很多人下意识觉得维也纳公约（1961）那个年代没什么通讯设备，所以条文不适用今天。这个观点不对。

1961 年的通讯生态其实非常丰富：

  • 电传打字机 Telex 已经在全球互联 20 多年
  • 各国大使馆国际短波通信无线电是标配
  • 美苏热线 1963 年要建
  • 各国情报机构搞 SIGINT 已经 30 年
  • 密码机如 KW-7、Hagelin C-52 已经服役多年

国际通讯规则更早——1865 年就有国际电报公约（马克思在伦敦写资本论的时候就有了）。所以 1961 年的维也纳公约只是把 100 年的既有惯例法典化。

第 27 条核心内容：

东道国应允许并保护使团为一切公务目的的自由通讯。使团可使用一切适当方式，包括外交信使和密码或暗号电报。

1961 年就承认了加密通讯权。还有：

  • 使团的官方信件不可侵犯
  • 外交邮袋不可开拆或扣留（这就是外交包，可以塞任何东西）
  • 但使团安装和使用无线电发报机，必须经过东道国同意

最后这条很关键。理论上中国可以拒绝美方在境内架设无线电发射设备，但实践中每次国事访问之前，美方先遣队都会跟中方协商一份地面规则清单，明确列出：

  • 要带哪些通讯设备
  • 用什么频段
  • 覆盖范围多大

中方批准 → 美方部署。

所以中方去封禁特朗普通讯，在公约语境下不是技术问题，而是条约问题。中方在他抵达之前就已经书面同意了美方的整套通讯部署。如果中方在这个情况下去干扰它，等于撕毁双边协定，违反习惯国际法——属于宣战级别的外交事件。

封禁的成本远高于任何可能拿到的收益。

八、反讽：美国总统是世界上受最严格网络管控的人

到这里答案已经很清楚：特朗普完全不用翻墙，可以畅通无阻地访问任何互联网。

但这不意味着他自由。 实际上他正在经受着比任何中国人都严格的网络管控和审查，只是管他的不是 GFW，是 NSA + WHCA + 美国宪法体制。

8.1 没人能给美国总统发威

  • 年薪 40 万美元固定，宪法第二条第一款规定任期内不得加薪不得减薪
  • 国会都改不了
  • 所以没有 KPI 考核扣工资这种东西

8.2 NSA 跟特朗普的拉锯战

2017 年第一任期初期，NSA 和 WHCA 给他的 iPhone 是完全砍掉所有第三方 App 的状态，包括 Twitter。

但他在 2016 年竞选期间养成了每天凌晨 3 点发推的习惯，上任后立刻表示"我要发推"。当时的白宫办公厅主任 Reince Priebus 和后来的 John Kelly 都试过限制他，根本拐不动。特朗普绕开障碍的办法：

  • 找助理发
  • 用其他设备发
  • 干脆拿起未经改装的私人 iPhone 自己发

NSA 这时候要做现实主义的决定：

  • 选项 A：拒绝预装 Twitter，让他用私人未改装 iPhone → 设备完全暴露
  • 选项 B：预装 Twitter 到改装机 → 设备本体安全，推文公开（反正本来就要公开）

最后选了 B，叫 Harm Reduction（伤害最小化）——NSA 局长签字的安全例外。

所有人都知道这是走形式：总统是 NSA 的最终上级，他想用什么 App，NSA 最多写一份措辞强硬的风险评估报告，他签字"我接受风险"就推翻了。

美国总统通过持续骚扰自己的安全部门，强迫他们让步给他装了一个 Twitter。 这是世界上最荒诞的"用户提需求"故事之一。

8.3 体制反向制裁总统

既然管不住，那安全部门怎么办？答案是反向制裁——政治学里有专门的研究框架叫 Adversarial Principal-Agent Problem：委托人和代理人利益不一致甚至敌对的时候，组织如何运转。

从匿名爆料和回忆录里看到的反制手段：

  • 有选择地呈递文件——重要但他可能冲动反应的事不告诉他
  • 延迟执行命令——前国防部长 Mattis 公开承认过
  • 从桌上偷走文件——前国家经济委员会主任 Gary Cohn 的著名事件：特朗普准备签一份退出美韩自贸协定的命令，被 Cohn 直接从办公桌上偷走，特朗普回来忘了这事，命令也就没签成（Bob Woodward 的书里有详细记载）
  • 操控会议议程——调整每日情报简报（PDB）的呈现顺序、长度、视觉风格。特朗普被发现更容易吸收"短的 + 有图的 + 有他名字的"简报
  • 隐形抗命——参联会主席 Mark Milley 那次特殊抗命就是典型

更典型的物理隔离：椭圆办公室 / 战情室 / 空军一号密谈区，都是按法拉第笼设计的，电磁屏蔽。手机进去就是一块砖。对总统本人没有任何例外。

既然劝不动你不要用手机，那就把你直接锁进一个连手机都用不了的房间里开会。

8.4 这种机制到底是好是坏

组织行为学里这是双面的：

  • 好的一面：组织对个人疯狂行为有缓冲，避免单一个体的灾难性决策
  • 坏的一面：未经选举授权的官僚系统反而限制经选举上台的总统，本身就违反民主原则

哪边赢看你站哪边。

九、PRA：所有通讯档案最终归人民

特朗普现在用的这些 iPhone 未来怎么处理？答案是 Presidential Records Act of 1978（PRA，总统记录法）。

这部法律是水门事件的直接产物。1981 年里根上任的时候正式生效，里根是第一位受 PRA 约束的总统。

核心规定：

  • 总统/副总统在任期间产生的所有公务记录都属于美国人民，不是私人物品
  • 总统没有销毁权
  • 任期结束的瞬间，所有记录的法律所有权自动转交给 NARA（National Archives and Records Administration，国家档案馆）
  • 加密档案和密钥同时移交给 NARA 的档案管理员，存入总统图书馆（每位总统都有自己的图书馆，奥巴马的在芝加哥）

9.1 那我们能不能看到？

根据 2009 年奥巴马签署的行政命令，机密分三级：

等级 自动解密时间
Confidential 10 年
Secret 25 年
Top Secret 50 年

特殊豁免（情报来源 / 密码学方法 / 核武器设计 / 活线人身份）可延迟到 75 年甚至更久。

所以特朗普第二任期的通讯档案：

  • 2029 年任期结束 → 移交 NARA
  • 2039–2054 年间普通分类陆续解密
  • 2079 年左右 Top Secret 级别解密

你和我这辈子大概率能看到特朗普这一周发给马斯克或库克的短信。2040 年代的历史学家会拿来写博士论文。

十、判断框架：消费级安全 ≠ 政府级安全

最后留一个判断框架：

当一个商业产品说"我们的安全级别和总统一样"，可以默认这句话不准确。要么夸大了，要么偷换了概念。

真正的政府级保密设备从不会进入消费级市场。因为他们的设计前提——

  • 空气供应链
  • 定制硬件
  • 专用网络
  • 合规审计

——和消费产品的设计前提——

  • 规模化量产
  • 广泛兼容
  • 用户自由

——从根本上是冲突的。

你买的 BlackBerry 和奥巴马用的那台 BlackBerry 几乎不是同一个东西（NSA 定制硬件套了一个黑莓外壳）。你用的 iPhone 和特朗普用的那两台 NSA 改装 iPhone 也不是同一个东西。

消费级和政府级之间，不存在连续光谱。它们是两个不同的物种。

结尾

5 月 15 日特朗普离开北京，他的两台改装 iPhone 跟他一起飞回华盛顿。记录这次访华的所有通讯数据会进入 NARA 国家总统图书馆，等待 2050 年代某一天被解密。

而我们，正在用各种方式翻墙看视频写博客的普通人，使用的工具——从快连到自建节点到 ShadowRocket——都不是为我们设计来翻越某座墙的。它们只是在这座墙允许我们存在的边界上，留下了一个临时性的缝隙。

缝隙什么时候关，不取决于我们，而取决于墙。

这个事实让人不舒服。但面对这个事实，是理解所有大规模系统的第一步。
